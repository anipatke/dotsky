# dotsky — v1.1 Release Plan

## Release goal

Close the gap between the shipped v1 and the PRD: fix the conformance defects found in the 2026-07-04 project review (ignored `--time` flag, broken Unicode detection on Linux, small-terminal corruption, inverted star priority, doubtful SIGTERM restoration), restore a fully green test suite so the publish gate works, bring the render loop into §9 compliance, and validate accuracy against a reference planetarium before handoff.

## Epic summaries

### E01: test-suite-integrity
Fix the cross-test state pollution that makes 3 tests fail in the full suite while passing in isolation (global resize-listener teardown in `App.tsx` unmount, never-unmounted test instances, module-level Unicode detection). Restores a trustworthy `prepublishOnly` gate — currently `npm publish` would fail its own precheck. Every later epic's quality gate depends on this.

### E02: conformance-defects
Fix the five behavioral deviations from the PRD found in review: `--time` is clobbered by live mode within one second; `detectUnicodeSupport()` ignores `LANG` so UTF-8 Linux terminals get ASCII symbols; terminals below 80x24 get a wrapped/corrupted sky instead of a warning screen; star density priority is inverted vs PRD §10; `process.exit(0)` on SIGTERM may skip Ink's terminal restoration. Plus removal of dead `terminal/screen.ts`.

### E03: render-loop-efficiency
Bring the render loop into PRD §9 compliance: celestial positions computed once per second instead of 15 times per second, render capped at fps, no redraws when nothing changed. Improve label layout to width-aware collision avoidance with right-edge clamping (§12).

### E04: release-validation
Packaging hygiene (`savepoint` out of runtime dependencies, `engines` field), accuracy spot-check of ≥3 bodies against Stellarium per the Phase 3 validation strategy, and a manual QA pass of the PRD §18 success criteria in a real terminal.

## Out of scope for this release

- Everything in the v1 out-of-scope list (constellations, search, mouse, city lookup, themes, export, telescope, deep-sky, AR)
- Alternative projections (stereographic/orthographic/azimuthal equidistant) — the §7 isolation seam already exists
- Reworking aspect correction to use the full viewport height (recorded as a known design smell; revisit with projection work)
- Expanding the star catalog beyond the current 97 stars
