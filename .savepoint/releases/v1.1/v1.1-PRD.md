# dotsky — v1.1 Release Plan

## Release goal

Close the gap between the shipped v1 and the PRD: restore a deterministic publish gate; fix the confirmed time, Unicode, small-terminal, density-priority, shutdown, and astronomy-failure behaviors; replace the continuous render timer with an event-driven pipeline; declare truthful Node.js compatibility; and validate accuracy and terminal behavior before handoff.

E01 test-suite-integrity

Tasks may run in parallel only inside an epic when their `depends_on` fields allow it. E04 manual QA is the final exit gate and cannot begin until every implementation task is complete.

## Epic summaries

### E01: test-suite-integrity
Fix the cross-test state pollution that makes 3 tests fail in the full suite while passing in isolation (global resize-listener teardown in `App.tsx` unmount, never-unmounted test instances, module-level Unicode detection). Restores a trustworthy `prepublishOnly` gate — currently `npm publish` would fail its own precheck. Every later epic's quality gate depends on this.

### E02: conformance-defects
Fix the confirmed behavioral deviations from the PRD: `--time` is clobbered by live mode; locale detection skips `LANG`; terminals below 80x24 render a corrupted clamped sky; density priority disagrees with §10; shutdown paths are fragmented; and astronomy exceptions are not handled at the application boundary. Remove dead `terminal/screen.ts` and use one idempotent shutdown path.

### E03: render-loop-efficiency
Replace the continuous FPS timer with event-driven React derivation: live time updates at 1 Hz, controls and resize update immediately, and an unchanged memoized viewport does not rerender when only the header clock changes. Deprecate `--fps` for compatibility in v1.1 rather than preserving a timer with no useful work. Improve label layout to full-extent collision avoidance with deterministic priority and viewport-safe placement.

### E04: release-validation
Packaging hygiene (`savepoint` out of runtime dependencies, Node `>=22` declared and documented), repeatable accuracy fixtures backed by a recorded reference comparison, and a manual QA matrix covering supported terminals, locales, signals, resize boundaries, offline behavior, and the packed global install.

## Out of scope for this release

- Everything in the v1 out-of-scope list (constellations, search, mouse, city lookup, themes, export, telescope, deep-sky, AR)
- Alternative projections (stereographic/orthographic/azimuthal equidistant) — the §7 isolation seam already exists
- Reworking aspect correction to use the full viewport height (recorded as a known design smell; revisit with projection work)
- Expanding the star catalog beyond the current 97 stars
- Removing the deprecated `--fps` flag (planned for the next breaking CLI release)
